# Аналитический сервис рынка труда для HR-специалистов

Сервис автоматически собирает вакансии с hh.ru, хранит их в базе данных
и предоставляет удобный интерфейс для анализа рынка труда.

---

## Что умеет сервис

- Собирать вакансии с hh.ru по ключевым словам и региону
- Фильтровать вакансии по зарплате, опыту и названию
- Показывать медианную зарплату и распределение по опыту
- Авторизоваться через личный кабинет hh.ru

---

## Первый запуск

### Шаг 1 — Авторизация

Откройте в браузере:
```
http://localhost:8080/auth/login
```

Войдите через аккаунт hh.ru. После успешного входа вернётесь на сервис.

### Шаг 2 — Сбор вакансий

В Swagger ([http://localhost:8080/docs](http://localhost:8080/docs)) найдите эндпоинт:

```
GET /vacancies/collect/
```

Укажите параметры:
- `text` — название позиции, например `менеджер по продажам`
- `area` — код региона (`1` — Москва, `2` — Санкт-Петербург)

Нажмите Execute. Сервис соберёт вакансии и сохранит в базу данных.

### Шаг 3 — Анализ

Используйте эндпоинты для работы с данными:

| Эндпоинт | Что делает |
|---|---|
| `GET /vacancies/sorted/` | Список вакансий с фильтрами |
| `GET /vacancies/stats/` | Медиана зарплат и распределение по опыту |
| `GET /vacancies/{id}/` | Детали конкретной вакансии |

---

## Обновление токена

Токен доступа действует 2 недели. Когда истечёт — вызовите:

```
GET /auth/refresh_token
```

Или пройдите авторизацию заново через `/auth/login`.

## Возможные проблемы

**Ошибка подключения к БД**
Убедитесь что Docker запущен и база данных поднята:
```bash
docker-compose up -d db
```

**Ошибка авторизации (403)**
Токен истёк. Обновите через `/auth/refresh_token` или войдите заново.

**Вакансии не собираются**
Проверьте что авторизация прошла успешно и токен сохранён в БД.
